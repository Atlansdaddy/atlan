What Atlan is
A cockpit for building software from your phone. One screen talks to Claude Code (a real agent that edits files, runs commands, and builds apps), previews what you're making, gives you a real terminal, runs autonomous agent fleets on budgets, schedules routines, compiles your Persona+ framework into working agent parts, ships installable APKs, and watches its own health. It runs entirely on this phone at 127.0.0.1 — nothing leaves unless you build the tunnel yourself, and the preflight gate holds you honest about when that's safe.


    
Chat — talking to engines
Project picker: every folder in /root with git or a package.json. Everything (chat, fleet, build, terminal) happens in the picked project.
Switcher groups: Claude Code models are AGENTS — they act. Codex/Gemini CLIs act too but full-auto (no per-tool cards). On-phone and cloud "brains" only talk — they will say so if you ask them to act.
Permission cards: when Claude wants to run something risky you get Allow/Deny. Deny is always safe — the agent is told why and works around it.
Session line: tap to copy claude --resume <id>; the same conversation continues in any terminal.
Preview context: console errors and 📸 snapshots attach themselves to your next message automatically.


    
Preview — watching your app
Enter a local dev-server URL (127.0.0.1 only — this is a security choice, not a limitation) and load. The cockpit proxies it and injects a tiny watcher: console output appears in the strip; errors queue for Claude's next turn with file:line; 📸 saves a real PNG that Claude reads with vision. HMR/live-reload passes through untouched.


    
Term — the escape hatch
A real shell in tmux session atlan-main. From Termux, tmux attach -t atlan-main mirrors the exact same screen both ways. Stored API keys are injected into its environment so CLIs auth the same as the cockpit. If the GUI ever annoys you, everything it does is doable here — by design.


    
Fleet — autonomous runs
Write a task, pick a profile + model + budget, spawn. Profiles are hard walls, not requests: Scout = read-only (write tools stripped at the CLI level, provable). Builder = files + bash, writes fenced to the project folder. Verifier = reads + runs checks, never edits. Budgets HALT: at the cap the run stops (checked between steps, so one step can overshoot a little before the halt — no unbounded spend); first turn costs ~35k tokens so 50k is the floor. Top-up: a halted run resumes its exact session with fresh budget — nothing is lost. Inbox: report cards survive restarts; chat gets a ping line; the Fleet tab badge counts unseen reports. Push: enable once, get real notifications with the app closed. KILL ALL: immediate, always available. Idle fleet = zero tokens, by construction.


    
Routines — scheduled agents
Every-N-minutes or daily-at-HH:MM. Each fire is a normal budgeted fleet run (same walls) labeled routine:<name> in the inbox. Missed-run rule: if the server was off past a slot, the routine is flagged MISSED and does nothing until you tap "run late" — a rebooting phone can never silently spend. Pause-all freezes the clock. A disabled routine keeps its config.


    
Builder — Persona+ compiled
Your 2023 framework is the native language here. Persona (NAME/FOCUS/BIO/SKILLS/NO_NOS/TEMPLATE/INSTRUCTIONS) → the system prompt; pick the fleet profile it runs under. Structured command → a typed tool: VARIABLES become JSON-schema parameters; TEMPLATE fields become a constrained JSON answer; the model literally cannot return the wrong shape on engines with constrained decoding. Checkers are deterministic assertions graded by code, never by a model: enum ∈, range, regex, ⊆ input-variable (no invented parts), max length, arithmetic formulas (total = qty*price). Harness: run any command on local/cloud engines, see every check pass/fail with evidence, and escalate failures to a Claude fleet run in one tap. Personas also plug into routines and (soon) chat.


    
Build — APK on demand
One button runs the proven pipeline: env.sh → web build with CAP_BUILD=1 → Capacitor sync → Gradle using the qemu shim that lets the x86-only aapt2 run on this ARM phone. Log streams live. Every APK gets a unique name + visible stamp (stale-cache dodge). Serve/install straight from the cockpit. Needs ~2.5GB free RAM — stop llama-server first if things get tight.


    
Doctor — health
Checks every fragile proot-boundary piece: JDK21, Android SDK, aapt2 shim, claude binary + auth, tmux, disk space, llama-server, and that the push service worker has NO fetch handler (the promise that stale caching is impossible). When a Termux update breaks something, this screen names it.


    
Keys & security
Keys: AES-256-GCM encrypted at rest (.keys.enc + 0600 secret), masked to last-4, never echoed. Env vars win over stored keys.
Auth: you set a password on first run (scrypt-hashed); the whole API, APK downloads and the live socket require an httpOnly, SameSite=Strict session cookie you get by logging in — it survives restarts, so no re-login. Failed logins are throttled. Automation (tests/CLI) can use a bearer header instead — never a URL. Change your password in Doctor → Session.
Preflight: the "safe to expose?" gate — loopback bind, password set, encrypted keys, no plaintext, gitignore coverage, no live tunnels. It goes green once you've set a password; exposure is still YOUR call and should go through Cloudflare Tunnel + Access, never a bare port.


    
Atlan himself
Mood is real state: calm teal = idle (idle is free). building shimmer = agents/builds active — orbiting lights count your running agents. alarmed orange pulse = doctor red or a run needs you. proud flash = something surfaced. He greets you by time of day, comments on events, and the whole cockpit dims to night water 22:00–06:30 — presence, not decoration.


    
Troubleshooting
Forgot your password? it's scrypt-hashed and can't be recovered — delete .fleet/auth.json (in the app's folder) and restart to set a new one on next load; your data (inbox, keys, routines) is untouched.
Server down? cd /root/atlan && node server/src/index.js (or nohup it). Doctor tab confirms.
Session died overnight? Android's phantom-process killer — known, documented, wake-lock helps; the durable stores (inbox, burn, routines, personas) mean nothing of record is lost.
Build fails on RAM? stop llama-server, retry.
Push never arrives? re-tap 🔔 (subscriptions rot when the browser clears data); Doctor asserts the worker is still push-only.
A routine missed its slot? that's the design — tap ▶ run late.
